# ReplayCS product completion board

Audit date: 2026-07-20

`Complete` means the stated P0 acceptance path is implemented; it does not mean the entire topic
roadmap is shipped. `Partial` means useful, tested functionality exists with a disclosed remaining
gap. These board labels preserve the pre-expansion product acceptance paths; they do not mean a
lesson satisfies the new 34-point operation-complexity contract. The
[curriculum completeness matrix](curriculum-completeness-matrix.md) is authoritative for that stricter
status, and currently marks no topic expansion-complete. A subject dashboard alone never counts as
functional.

## Verified release baseline

- Public production: <https://replaycs.vercel.app>
- Safe health endpoint: <https://replaycs.vercel.app/api/health>
- Functional routes: onboarding, progress, Judge Demo, Challenge Arena, Binary Search, Sorting Arena,
  Graph Explorer, SQL Query Pipeline, CPU Scheduling Arena, and Packet Journey.
- Foundation checkpoint `28d4f6b` adds the `/complexity` release candidate: 15 documented growth
  families, nine safe chart curves, and two honestly bounded operation analyses. Do not treat it as
  deployed on the stable domain until the checkpoint is merged and production-verified.
- Production reports `aiConfigured: true`, and a structured `source: "ai"` mentor response is
  verified; deterministic fallback keeps the mentor usable without `OPENAI_API_KEY` or during an
  upstream failure.
- Local release gate: `npm run check`, `npm run lint`, `npm run test`, `npm run build`, and Chromium
  Playwright. GitHub Actions encodes the same gate and is green for the release and live-mentor fix.

## Release-close checklist

1. Run the complete local gate on the final integrated commit.
2. Push the focused product-polish commit and merge without rewriting history.
3. Confirm GitHub Actions and Vercel Production both use the final `main` SHA.
4. Rerun `/api/health`, production smoke, Judge Demo, mobile viewport, and browser-console checks.
5. Verify `/complexity` on Preview and Production, including case-matched Binary Search work and
   the absence of unsupported trace tabs.
6. Regenerate screenshots from the final deployment and rehearse the under-three-minute script.
7. Replace `CODEX_SESSION_ID=ADD_FROM_CODEX_FEEDBACK_COMMAND` only with the actual `/feedback` value.
8. Review [known-limitations.md](known-limitations.md), then create the final annotated release tag.
